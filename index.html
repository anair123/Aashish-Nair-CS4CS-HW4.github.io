I will write a paragraph for each methodology of SDLC. There are many of
them but I will suggest a few. My suggestion and reasons are as follows.

  1. A) The Waterfall model is a model that comprises 6 steps: requirements, design, implementation, test, installation, and maintanence. The requirements phase defines the needed information, the design phase defines the algorithms and software architecture, the implementation phase entails writing the source code, the testing phase entails performing testing, the installation phase entails deployment, and the maintainance phase entails maintaining the system. It is not complex and is suited for inexperienced staff. This model is useful when the requirements are well known and the product definition is stable.
  2. B) The V-Shaped model is a variation of the waterfall model in which the development and testing are performed parrelelly. It is easy to use and focuses on verification and validation in the early stages. However, it does not handle dynamic changes in requirements in well. This model is suited for systems needing high reliabilty.
  3. C) The Structured Evolutionary Prototyping model entails building prototypes continuously until one reaches the standards of the final product. It begins with developing a prototype based oon a high-level paper model. The designer shows the prototype and modifies it based on the user feedback. The designer continues building prototypes until the users are satisfied. This model allows constant feedback from customers. Thus, the model is suited for short-lived demonstrations and the requirements classification stage of a waterfall model.
  4. D) The Rapid Application Development (RAD) model comprises 4 steps: requirement planning phase, user description phase, construction phase, and cutover phase. This model uses a time-box approach to improve speed and productivity. The RAD model is suited for projects that can be time-boxed.
  5. E) The Incremental model entails constructing a partial implementation of a total system and adding functionalities until the final product is ready. This model allows a faster initial product delivery, with all the major functions being priotized.
  6. F) The spiral model entails performing 4 phases (or quadrant) iteratively. The first quadrant is about determining objectives and constraints. The second quadrant is about identifying and resolving risk. The third quadrant focuses on developing the product, which includes creating designs and developing code. The final phase is for planning the next phase, including project and test plans. This model puts a lot of emphasis on risk assessment. It also allows frequent user feedback. Thus, it is ideal for projects that are high risk or are subject to changes.
  7. G) The Extreme Programming (XP) model entails small teams building software with everchanging requirements. The model emphasises the use of code to complete deliverables and communicate among teammates. The model enforces pair programming, where two programmers write production code in one machine.
  8. I) The Scrum methodology conducts software development in stages of development called sprints. Each sprint is dedicated to bringing the team closer to their goal. This methodology has teams starting every day with a short meeting, in which the team can evaluate its progress. This methodology is suited for fast-pace environments and allows teams to modify requirements throughout the project.

  3. C) The Kanban methodology allows teams to track their tasks by visualizing their project with a Kanban board. The board separates tasks under three columns: "to do", "doing", and "done." The methodology is ideal for teams that work with changing requirements and seek a fast product delivery.
  4. D) The Crystal methodology focus on individuals rather than processes and tools. This is one of the more flexible methodoligies since it allows teams the freedom to work as they see fit. This is more suited for teams working on short-term projects in a single workspace.

I have used CSS in this webpage and there are 3 ways to include css in a
webpage as follows.

  1. A) CSS can be integrated with in text styling. This entails using the style attribute in a given HTML tag (e.g., p tag) and assigning values to the css attributes (e.g., font-size).
  2. B) CSS can be added with internal CSS, which entails using the style tag inside the head tag of the HTML file.
  3. C) CSS can be added using external CSS, which entails creating a seperate .css file that contails all css content and linking it to the original HTML file.

  1. 1. CSS improves the the design and appeal of the webpage with its styling options.
